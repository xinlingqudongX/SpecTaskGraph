1. 项目概述
项目名称： 工作流程图编辑器 (Workflow Diagram Editor)

核心目标： 提供一个可视化编辑平台，使产品经理、系统架构师和开发人员能够清晰定义业务功能节点与依赖关系，并将其导出为 AI 可读取的结构化数据，驱动 AI Agent 自动实现或生成相应的业务代码。

目标用户： 产品经理、业务架构师、研发人员。

2. 核心业务流程
节点定义： 用户在画布上拖拽创建业务节点（如：接口定义、数据验证、业务逻辑处理、数据库读写等）。

依赖编排： 用户通过连线定义节点间的执行顺序和数据流转依赖。

属性配置： 为每个节点配置详细的业务规则、输入/输出参数要求。

图谱导出： 将绘制好的流程图导出为机器和 AI 友好的结构化格式。

AI 消费与执行： AI 引擎解析导出的图数据，根据依赖树逐个突破，生成对应的代码或直接执行工作流。

3. 核心架构与技术选型建议
为了确保系统的可维护性和 AI 解析的高效性，建议在底层架构上做如下设计：

服务端架构： 可以采用 TypeScript + NestJS，并结合服务端渲染（SSR）来构建。这样能够统一技术栈，简化多框架维护的成本，同时保证后端服务的严谨性。

数据流转与存储格式： 导出的工作流程节点不建议使用传统的扁平文件（如纯 Markdown 文档），而是将其抽象为**知识图谱（Knowledge Graph）**或有向无环图（DAG）结构。使用图数据结构（或图数据库）存储，能够让 AI 更精准地理解复杂业务逻辑中的上下游关系和边界条件。

AI 引擎交互设计： 在 AI 自动实现功能的环节，为避免在复杂的业务流中每次请求都传递完整的业务逻辑树（这会导致巨大的 Token 消耗和上下文丢失），建议在 AI 执行引擎中引入状态管理机制。通过动态插槽填充（Dynamic Slot Filling），AI 在处理特定节点时，只需加载该节点及其直接前驱/后继节点的上下文即可。

4. 功能模块划分
画布编辑模块： 支持拖拽、连线、节点编组、缩放、撤销/重做（可考虑引入类似 AntV X6 或 React Flow 的底层库）。

节点配置面板： 动态表单生成，根据不同的业务节点类型，提供不同的属性输入（如校验规则、SQL 预览、API 接口定义）。

编译与导出模块： 负责将画布上的视觉 JSON 数据，编译转换为 AI Agent 所需的图结构指令集。

项目与协作管理： 团队空间、版本控制、多角色协同编辑。
